클라우드 제공자의 필수 topic 구조 때문에 MQTT topic 구조는 엄격히 정의되어 있지 않습니다.
클라우드 기반 MQTT 브로커의 경우 topic 구조를 개별적으로 맞춰야 할 수 있으나, 제안된 구조를 대략적으로 따르는 것이 좋습니다.
다음 절에서 정의한 topic 이름은 필수입니다.

로컬 브로커의 경우 MQTT topic 수준은 다음과 같이 제안됩니다.

**interfaceName/majorVersion/manufacturer/serialNumber/topic**

예:

```
vda5050/v3/KIT/0001/order
```

MQTT Topic Level | Data type | Description
---|---|---
interfaceName | string | 사용되는 인터페이스의 이름
majorVersion | string | VDA 5050 권고의 주 버전 번호, 앞에 "v"를 붙임
manufacturer | string | 모바일 로봇 제조사
serialNumber | string | 다음 문자로 구성된 모바일 로봇 고유 일련번호: <br>A-Z <br>a-z <br>0-9 <br>_ <br>. <br>: <br>-
topic | string | Topic(예: order 또는 state) [4.4 Topics for Communication](/spec/4/4.3#43-topics-for-communication) 절 참조

>표 1 제안된 MQTT topic 수준 설명

`/` 문자는 topic 계층을 정의하는 데 사용되므로, 위 필드 어디에도 사용해서는 안 됩니다.

The MQTT topic structure is not strictly defined due to the mandatory topic structure of cloud providers.
For a cloud-based MQTT broker the topic structure might have to be adapted individually, but it should roughly follow the proposed structure.
The topic names defined in the following sections are mandatory.

For a local broker the MQTT topic levels are suggested as followed:

**interfaceName/majorVersion/manufacturer/serialNumber/topic**

Example:

```
vda5050/v3/KIT/0001/order
```

MQTT Topic Level | Data type | Description
---|---|---
interfaceName | string | Name of the used interface
majorVersion | string | Major version number of the VDA 5050 recommendation, preceded by "v"
manufacturer | string | Manufacturer of the mobile robot.
serialNumber | string | Unique mobile robot serial number consisting of the following characters: <br>A-Z <br>a-z <br>0-9 <br>_ <br>. <br>: <br>-
topic | string | Topic (e.g., order or state) see Section [4.4 Topics for Communication](/spec/4/4.3#43-topics-for-communication)

>Table 1 Explanation of suggested MQTT topic levels

Since the `/` character is used to define topic hierarchies, it shall not be used in any of the aforementioned fields.
Wildcard characters `+` and `#` as well as the character `$` that is reserved for broker internal topics should not be used either.
