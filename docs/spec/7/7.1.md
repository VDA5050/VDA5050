### 7.1.1 Optional fields

<BilingualSection>
  <template #ko>
<div>

변수가 선택 사항으로 표시되면, 해당 변수가 특정 상황에 적용되지 않을 수 있으므로 송신자에게 선택 사항입니다(예: fleet control이 모바일 로봇에 Order를 보낼 때, 일부 모바일 로봇은 궤적을 스스로 계획하며 Order의 `edge` 객체 안의 `trajectory` 필드는 생략될 수 있음).

모바일 로봇이 이 프로토콜에서 선택 사항으로 표시된 필드를 포함하는 메시지를 수신하면, 모바일 로봇은 그에 맞게 동작해야 하며 해당 필드를 무시해서는 안 됩니다.
모바일 로봇이 지원되지 않는 매개변수 때문에 Order를 처리할 수 없으면, 오류 유형 'UNSUPPORTED_PARAMETER' 및 오류 수준 'CRITICAL'로 통신하고 Order를 거부해야 합니다.

Fleet control은 모바일 로봇이 지원하는 선택 필드만 보내야 합니다.

예: Trajectory는 선택 사항입니다.
모바일 로봇이 trajectory를 처리할 수 없으면, fleet control은 해당 모바일 로봇에 trajectory를 보내지 않아야 합니다.

모바일 로봇은 어떤 선택 매개변수가 필요한지 모바일 로봇 `factsheet` 메시지를 통해 통신해야 합니다.

</div>
  </template>
  <template #en>
<div>

If a variable is marked as optional, it is optional for the sender as the variable might not be applicable in certain cases (e.g., when the fleet control sends an order to a mobile robot, some mobile robots plan their trajectory themselves and the field `trajectory` within the `edge` object of the order can be omitted).

If the mobile robot receives a message that contains a field which is marked as optional in this protocol, the mobile robot is expected to act accordingly and shall not ignore the field.
If the mobile robot cannot process the order due to an unsupported parameter, it shall communicate this with an error of type 'UNSUPPORTED_PARAMETER' and error level 'CRITICAL' and to reject the order.

Fleet control shall only send optional fields that the mobile robot supports.

Example: Trajectories are optional.
If a mobile robot cannot process trajectories, fleet control shall not send a trajectory to the mobile robot.

The mobile robot shall communicate which optional parameters it needs via a mobile robot `factsheet` message.

</div>
  </template>
</BilingualSection>

### 7.1.2 Permitted characters and field lengths

<BilingualSection>
  <template #ko>
<div>

모든 통신은 설명의 국제적 적용을 위해 UTF-8로 인코딩됩니다.
ID에는 다음 문자만 사용할 것을 권장합니다:

A-Z a-z 0-9 _ - . :

최대 메시지 길이는 정의되어 있지 않지만, MQTT 프로토콜 사양 및 Factsheet에 정의된 기술적 제약에 의해 제한될 수 있습니다.

모바일 로봇의 메모리가 수신 Order를 처리하기에 부족하면, Order를 거부하고 오류 유형 'INSUFFICIENT_MEMORY', 오류 수준 'URGENT'로 보고해야 합니다.

최대 필드 길이, 문자열 길이 또는 값 범위의 일치 여부는 통합자(integrator)에게 달려 있습니다.

통합을 용이하게 하기 위해, 모바일 로봇 공급업체는 [7.10 Implementation of the factsheet message](/spec/7/7.10#710-implementation-of-the-factsheet-message) 절에 상세히 설명된 모바일 로봇 Factsheet를 제공해야 합니다.

</div>
  </template>
  <template #en>
<div>

All communication is encoded in UTF-8 to enable international adaption of descriptions.
The recommendation is that IDs should only use the following characters:

A-Z a-z 0-9 _ - . :

A maximum message length is not defined, but limited by the MQTT protocol specification and possibly by technical constraints defined by the factsheet.

If a mobile robot's memory is insufficient to process an incoming order, it shall reject the order and report an error of type 'INSUFFICIENT_MEMORY' with error level 'URGENT'.

The matching of maximum field lengths, string lengths or value ranges is up to the integrator.

For ease of integration, mobile robot vendors shall supply a mobile robot factsheet that is detailed in Section [7.10 Implementation of the factsheet message](/spec/7/7.10#710-implementation-of-the-factsheet-message).

</div>

</BilingualSection>

Enumerations shall be written in uppercase, using an underscore to separate words, e.g., 'EXAMPLE_ENUMERATION'. These values are enclosed in single quotation marks in the document.
This includes keywords such as in the `actionStatus` field ('WAITING', 'FINISHED', etc.).
An extensible enum includes but is not limited to the predefined values for the paramter.

### 7.1.4 JSON data types

<BilingualSection>
  <template #ko>
<div>

가능한 한 JSON 데이터 형식을 사용해야 합니다.
따라서 부울 값은 열거형('TRUE', 'FALSE')이나 매직 넘버가 아니라 "true" 또는 "false"로 인코딩합니다.
수치 데이터 형식은 형식과 정밀도로 지정합니다(예: float64 또는 uint32). IEEE 754의 NaN, infinity 같은 특수 숫자 값은 지원하지 않습니다.

</div>
  </template>
  <template #en>
<div>

Where possible, JSON data types shall be used.
A Boolean value is thus encoded by "true" or "false", not with an enumeration ('TRUE', 'FALSE') or magic numbers.
Numerical data types are specified with type and precision, e.g., float64 or uint32. Special number values from the IEEE 754 like NaN and infinity are not supported.

</div>
  </template>
</BilingualSection>
