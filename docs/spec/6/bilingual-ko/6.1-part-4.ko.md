Order가 거절되어야 하는 경우는 여러 가지입니다.
이 시나리오는 그림 8에 나와 있으며 아래에 설명합니다.


#### 6.1.4.1 Mobile robot receives a malformed order

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 'VALIDATION_FAILURE' 유형·'WARNING' 수준 오류를 보고해야 합니다.
3. 이 경고는 모바일 로봇이 새 Order를 수락할 때까지 보고되어야 합니다.

#### 6.1.4.2 Mobile robot receives an order with optional fields it cannot use

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 'UNSUPPORTED_PARAMETER' 유형·'CRITICAL' 수준 오류와 잘못된 필드를 `errorReferences`로 보고해야 합니다.
3. 이 오류는 모바일 로봇이 새 Order를 수락할 때까지 보고되어야 합니다.

#### 6.1.4.3 Mobile robot receives an order with actions it cannot perform

예:

- 최대 승강 높이보다 높은 승강
- 스트로크가 없는데 승강 동작 등

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 'INVALID_ORDER_ACTION' 유형·'WARNING' 수준 오류와 잘못된 필드를 `errorReferences`로 보고해야 합니다.
3. 이 경고는 모바일 로봇이 새 Order를 수락할 때까지 보고되어야 합니다.

#### 6.1.4.4 Mobile robot receives an order with the same orderId, but a lower orderUpdateId than the current orderUpdateId

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 이전 Order를 버퍼에 유지합니다.
3. 모바일 로봇은 'OUTDATED_ORDER_UPDATE' 유형·'WARNING' 수준 오류를 보고해야 합니다.
4. 모바일 로봇은 이전 Order 실행을 계속합니다.
5. 이 경고는 모바일 로봇이 새 Order를 수락할 때까지 보고되어야 합니다.

#### 6.1.4.5 Mobile robot receives an order with the same orderId and same orderUpdateId as the current orderUpdateId

예:

- 플릿 제어가 해당 `orderUpdateId`를 포함한 State 메시지를 아직 받지 못해 Order를 다시 보냄.

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 이전 Order를 버퍼에 유지합니다.
3. 메시지 내용에 따라 보고:
   - 새 Order 내용이 이전과 같으면 모바일 로봇은 새 Order를 무시합니다.
   - 내용이 다르면 모바일 로봇은 'SAME_ORDER_UPDATE_ID' 유형·'WARNING' 수준 오류를 보고해야 합니다.
4. 모바일 로봇은 이전 Order 실행을 계속합니다.
5. 이 경고는 모바일 로봇이 새 Order를 수락할 때까지 보고되어야 합니다.

#### 6.1.4.6 Mobile robot receives an order with orderId different to the orderId of an active order

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 이전 Order를 버퍼에 유지합니다.
3. 모바일 로봇은 'OTHER_ORDER_ACTIVE' 유형·'WARNING' 수준 오류를 보고해야 합니다.
4. 모바일 로봇은 이전 Order 실행을 계속합니다.
5. 이 경고는 모바일 로봇이 새 Order를 수락할 때까지 보고되어야 합니다.

#### 6.1.4.7 Mobile robot receives an order with the start node being out of range

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 'START_NODE_OUT_OF_RANGE' 유형·'WARNING' 수준 오류를 보고해야 합니다.
3. 이 경고는 모바일 로봇이 새 Order를 수락할 때까지 보고되어야 합니다.

#### 6.1.4.8 Mobile robot receives an order with at least one node not being reachable

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 'NO_ROUTE_TO_TARGET' 유형·'WARNING' 수준 오류를 보고해야 합니다.

#### 6.1.4.9 Mobile robot receives an order while in an operating mode that does not allow new orders

처리:

1. 모바일 로봇은 새 Order를 내부 버퍼에 넣어서는 안 됩니다.
2. 모바일 로봇은 'MOBILE_ROBOT_NOT_AVAILABLE' 유형·'WARNING' 수준 오류를 보고해야 합니다.
3. 이 경고는 모바일 로봇이 새 Order를 허용하는 운전 모드가 될 때까지 보고되어야 합니다.

#### 6.1.4.10 Mobile robot receives an order containing nodes with unknown mapId

2. 모바일 로봇은 'UNKNOWN_MAP_ID' 유형·'WARNING' 수준 오류를 보고해야 합니다.
3. 이 경고는 모바일 로봇이 새 Order를 수락할 때까지 보고되어야 합니다.
